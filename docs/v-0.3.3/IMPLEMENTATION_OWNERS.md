# Implementation Owners

Status note. This document maps the target-state model in `docs/v-0.3.3/SYSTEM_MODEL.md` to the smallest current code owners expected to absorb the change. The active runtime remains documented under `docs/v-0.3.2`. To keep the owner map executable, current trace surfaces and future target contracts are separated below.

| Model Section | Current Code Owner | Current Trace Anchor | Target Contract / Convergence Note |
| --- | --- | --- | --- |
| Inputs and branch roles | `requirements/normalization.py`, `models.py` | `SearchRunBootstrapArtifact.input_truth`, `SearchRunBootstrapArtifact.requirement_sheet`, `FrontierNode_t` | Add `FrontierNode_t.branch_role` and `FrontierNode_t.root_anchor_frontier_node_id` as runtime-visible fields. |
| Bootstrap and root-anchor initialization | `bootstrap_llm.py`, `bootstrap_ops.generate_bootstrap_output`, `bootstrap_ops.initialize_frontier_state` | `BootstrapOutput.frontier_seed_specifications`, `SearchRunBootstrapArtifact.frontier_state` | Ensure round-0 emits at least one root anchor and preserves anchor lineage for every bootstrap seed. |
| Scoring policy freeze and rerank query | `bootstrap_ops.freeze_scoring_policy`, `rerank_text.build_rerank_query_text` | `SearchRunBootstrapArtifact.scoring_policy`, `ScoringPolicy.rerank_query_text` | Keep policy freeze stable while target output and repair semantics evolve. |
| Runtime entry and phase progression | `runtime/orchestrator.py`, `runtime_budget.build_runtime_budget_state`, `api.py` | `SearchRunBundle`, `run_dir`, `SearchControllerContext_t.runtime_budget_state` | Keep one shared phase owner while target stop semantics and reviewer output surface expand. |
