# Task: Phase 2.3B.2 Reflection Schema Slimming

## Goal

在 Phase 2.3B.1 finalizer draft slimming 完成后，精简 reflection 的模型输出和持久化 advice：

- 删除 `strategy_assessment`
- 删除 `quality_assessment`
- 删除 `coverage_assessment`
- 删除 keyword/filter advice 上的 persisted `critique`
- 保留结构化 keyword/filter advice、stop recommendation 和 runtime-built `reflection_summary`

业务目标是减少 reflection structured-output pressure，同时不改变 retrieval strategy、CTS filters、controller stop guidance、ranking semantics、eval semantics 或 controller 可用的反思记忆。

## Why Now

Phase 2.3 主计划要求 2.3B 同时覆盖 reflection 和 finalizer，但 reflection 会影响下一轮 controller/query/filter guidance，风险高于 finalizer。因此本计划排在 `docs/plans/active/phase-2-3b-1-finalizer-draft-slimming.md` 之后独立执行。

仓库证据：

- `src/seektalent/models.py` 中 `ReflectionAdviceDraft` 当前要求三段 prose assessment。
- `src/seektalent/models.py` 中 persisted `ReflectionAdvice` 也保存三段 assessment。
- `ReflectionKeywordAdvice` / `ReflectionFilterAdvice` 当前保存 `critique`，但 `src/seektalent/reflection/critic.py` 已能从结构化 advice runtime-build keyword/filter summary。
- `src/seektalent/runtime/context_builder.py` 的 `_reflection_summary()` 只传 `decision`、`stop_reason`、`reflection_summary` 给 controller summary view。
- `src/seektalent/runtime/orchestrator.py` 的 `round_review.md` 仍渲染三段 assessment heading/text。
- `src/seektalent/prompts/reflection.md` 当前要求返回 `ReflectionAdvice`，并在文字上提到 critique/prose。
- `tests/test_reflection_contract.py`、`tests/test_controller_contract.py`、`tests/test_context_builder.py`、`tests/test_runtime_audit.py`、`tests/test_runtime_state_flow.py` 仍使用三段 assessment 和 critique fixtures。

## Non-goals

- 不执行 Phase 2.3A.1 W&B suite/report correction。
- 不执行 Phase 2.3A no-eval artifact size gate closure。
- 不改 finalizer；本计划只能在 finalizer draft slimming 已完成后执行。
- 不改 scoring schema，不删除 `evidence`、`confidence`、`strengths`、`weaknesses`。
- 不改 retrieval/query compiler/query planner/CTS client/stop guidance/ranking sort key。
- 不改变 controller public decision schema。
- 不删除 `reflection_summary`，不让 controller 失去 compact reflection memory。
- 不引入 reflection discovery agent、verifier、fallback chain、retry chain 或新服务。

## Done Criteria

- `ReflectionAdviceDraft` 不再包含 `strategy_assessment`、`quality_assessment`、`coverage_assessment`。
- Persisted `ReflectionAdvice` 不再包含 `strategy_assessment`、`quality_assessment`、`coverage_assessment`。
- `ReflectionKeywordAdvice` / `ReflectionFilterAdvice` 不再持久化 `critique`。
- `materialize_reflection_advice()` 继续从 structured advice 生成 deterministic `reflection_summary`。
- `round_review.md` 保留一个 business-readable reflection summary 和 continue/stop decision。
- `round_review.md` 不再包含 `Strategy assessment`、`Quality assessment`、`Coverage assessment`。
- Controller context 仍能看到 previous reflection summary and latest structured advice。
- Focused tests 和 full tests 通过。
- 如执行真实 replay，则只跑 4-row acceptance subset，不跑 full benchmark。

## Current Reality

Observed behavior:

- `ReflectionAdviceDraft` requires three prose fields before any structured advice.
- `materialize_reflection_advice()` copies those prose fields into persisted `ReflectionAdvice`.
- `ReflectionKeywordAdvice.critique` is runtime-generated by `_keyword_critique()` and copied into `reflection_summary`.
- `ReflectionFilterAdvice.critique` exists on the model but is not currently populated by `materialize_reflection_advice()`.
- `round_review.md` currently renders `Reflection summary`, `Strategy assessment`, `Quality assessment`, `Coverage assessment`, and `Reflection decision`.
- `context_builder._reflection_summary()` only forwards `reflection_summary`, not the three assessment fields.

Known invariants:

- Reflection advice may guide later query-term pool updates through structured keyword advice.
- Reflection advice may guide later filter handling through structured filter advice.
- Controller should continue to receive compact previous reflection memory.
- Stop reason validation remains: `suggested_stop_reason` is required when `suggest_stop=true` and must be null when `suggest_stop=false`.

## Target Behavior

- Reflection model outputs only structured next-round advice plus stop recommendation.
- Runtime constructs one deterministic `reflection_summary` from:
  - round number;
  - unique new candidate count;
  - keyword activation/keep/deprioritize/drop advice;
  - filter keep/drop/add advice;
  - stop/continue decision.
- Persisted `reflection_advice.json`, `search_diagnostics.json`, controller context summaries, and `round_review.md` rely on `reflection_summary` and structured advice, not on three prose assessment fields.
- Business trace remains readable without extra UI/API work.
